데이터 모델 : 데이터 모형화 표현, 효율적으로 다루기 위한 모형 명세서
- 데이터 타입 집합   - 연산자 집합(데이터 처리)   - 무결성 규칙 집합(일관성 유지)
데이터 구성요소
- 개체(entity) : 식별 가능한 유형/무형적인 사물, 하나 이상 자신의 속성을 가진다
- 애트리뷰트(attribute) : 개체의 속성 / 각 애트리뷰트는 개체의 타입을 나타내고, 동일한 타입의 개체 집합은 해당 애트리뷰트들을 공유
예) 개체 타입 : 학생 / 학번, 이름, 지도교수, 소속학과 : 애트리뷰트
- 도메인(domain) : 각 애트리뷰트가 취할 수 있는 값의 집합
- 스키마(schema) : 데이터베이스를 구성하는 데이터 개체, 속성, 관계 및 데이터 조작 시 데이터 값들이 갖는 제약 조건 등 정의
사용자의 관점에 따라 외부 스키마, 개념 스키마, 내부 스키마로 나눠진다
- 관계 테이블 : 열(애트리뷰트)과 행(튜플)으로 이루어져 있다 
관계성 : 내부관계성, 외부관계성
데이터 무결성 : 데이터에 결함이 없는 성질
- 키(key) : 튜플을 유일하게 식별하기 위해 구성된 애트리뷰트들의 집합
- 후보키(Candidate Key) : 키 식별에 필요한 최소한의 애트리뷰트
- 기본키(Primary Key) : 하나의 테이블에 후보키가 둘 이상 있는 경우에 그 중 하나
- 대체키(Alternate Key) : 나머지 후보키
- 왜래키(Foregin Key) : T1 테이블의 기본키를 T2 테이블에서 이용하고 할 때 설정하여 사용하는 키
관계 모델의 두 가지 무결성 규칙
- 개체 무결성 규칙 : 기본키를 구성하는 애트리뷰트들은 어떤 것도 null 값을 가질 수 없다
- 참조 무결성 규칙 : T1에 있는 모든 외래키 값은 T2의 대응되는 기본키값과 같거나 null 값이어야 한다
                                    만일 null 값이면 참조할 튜플이 아직 결정되니 않은 것으로 해석
관계형 연산자 집합
- 선택 연산 : 주어진 조건에 만족하는 튜플 결과
- 투영 연산 : 필요한 애트리뷰트로만 이루어진 튜플 결과
- 카티션 곱 연산 : 연산에 참여하는 두 테이블 내 튜플들을 단순히 연접
- 자연 조인 연산 : 카티션 곱 결과에서 중복되는 애트리뷰트들 제거

2장 관계형 데이터베이스 설계
개념적 데이터 모델 : 실세계의 데이터 범위를 분명히 하고 개념적으로 이해하기 쉽게 나타낸 모델
- 개체-관계모델 : 실세계를 개체와 개체들 간의 연관성을 나타내는 관계성으로 모델링
